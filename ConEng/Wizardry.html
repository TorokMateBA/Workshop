1. SZEKCIÓ: Alapozás és Definíciók

Mi az a Context Engineering?

  • Ez a gyakorlat arra irányul, hogy a specifikus ágens-rendszer lépéseihez a lehető legkisebb mennyiségű, fókuszált kontextust biztosítsuk a feladat elvégzéséhez.
  • A hagyományos „Prompt Engineering” valójában a kontextus-mérnökség egy részhalmaza.

Az Ágens-rendszerek Topológiája:

  • Az ágens-rendszer különböző mintázatokkal összekapcsolt LLM-hívások topológiája (iterációs hurkok vagy szekvenciális minták).
  • Minden LLM-csomópont kimenete befolyásolja a rendszer következő lépéseit.

A minőség záloga:

  • Az ágens minősége csak annyira jó, amennyire az egyes lépéseknél az LLM-nek átadott kontextus.
  • A végtelenül táguló kontextusablakok nem jelentenek megoldást az alapvető problémákra.

2. SZEKCIÓ: A túlzott adatmennyiség veszélyei

A modell összezavarodása (Confusion Modes):

  • Context Poisoning: Hallucinációk kerülnek a kontextusba.
  • Context Distraction: A túl sok adat elnyomja a modell tanítását.
  • Context Confusion: Felesleges információk befolyásolják a választ.
  • Context Clash: A kontextus részei ellentmondanak egymásnak.

Teljesítménybeli korlátok:

  • Véges figyelmi keret (Attention Budget): Minden token versenyzik a modell figyelméért.
  • Precision Drop: A kontextus növekedésével csökken a precizitás és gyengül a következtetési képesség.
  • „Lost-in-the-middle” & „Needle in a haystack”: A modell hajlamos elveszíteni a középen lévő információkat, vagy nem találja meg a fontos részletet a zajban.

Operatív és biztonsági kockázatok:

  • Displacement: A túl sok adat kiszoríthatja a kritikus rendszerutasításokat.
  • Context Drift: Hosszú interakciók során a modell elveszíti a fókuszt az eredeti szándékról.
  • Prompt Injection: A külső forrásokból (pl. URL) beolvasott adatok rosszindulatú utasításokat tartalmazhatnak.
  • Költség és Latencia: A több token növeli a válaszadási időt és a működési költségeket.

3. SZEKCIÓ: A kontextus típusai

1. Rendszerutasítások (System Prompt):

Meghatározzák a szerepkört, viselkedést és határokat. Korlátok: illeszkedési (alignment) problémák és az injektálás kockázata.

2. Felhasználói Prompt (User Prompt):

A közvetlen kérés. Kihívás: a valódi szándék megértése több lépésből álló feladatoknál.

3. Beolvasott adatok (Retrieved Context):

Külső forrásokból (RAG) származó adatok. Kihívás: a legrelevánsabb információk kiválasztása a hatalmas tudásbázisból.

4. Rövid távú memória (State):

A „munkamemória”, az aktuális session legfrissebb állapota és előzményei. Korlátja a véges kontextusablak.

5. Hosszú távú memória:

  • Epizodikus: Korábbi interakciók és akciók.
  • Szemantikus: Külső ismeretek és az ágens önismerete.
  • Procedurális: Rendszerutasítások, eszközök sémái és korlátok.

6. Eszközök (Tools):

Eszközdefiníciók (JSON sémák) és az eszközök kimenetei. Kihívás: a definíciók rengeteg tokent fogyasztanak (90 eszköz > 50 000 token).

7. Strukturált kimenet (Structured Output):

JSON vagy SQL formátumok biztosítása a láncolt feladatokhoz. Segítő eszközök: Instructor könyvtár.

4. SZEKCIÓ: Megoldási minták (Context Engineering Patterns)

Pattern 1: Progressive Disclosure (Fokozatos közzététel):

  • Az információkat relevanciájuk alapján szintezzük (Discovery -> Activation -> Execution).
  • Agent Skills: Markdown fájlok, ahol csak a név és leírás töltődik be induláskor, a teljes leírás csak szükség esetén.

Pattern 2: Context Compression (Tömörítés):

  • Sliding Window: Csak az utolsó N kört tartja meg nyersen, a többit LLM-mel összegezi.
  • Szelektív megtartás: Az eszközhívások ritmusát és a hibaüzeneteket (error traces) érdemes megőrizni.

Pattern 3: Context Routing (Irányítás):

A kérést még a kontextusba kerülés előtt osztályozza. Típusai: LLM-alapú, hierarchikus vagy szabályalapú irányítás.

Pattern 4: Retrieval Evolution (Fejlett RAG):

  • Agentic RAG: Az ágens kontrollálja a keresési stratégiát.
  • Graph RAG: Kapcsolati összefüggések keresése dokumentumok között.
  • Self-RAG: A modell kritikusan értékeli saját kimeneteit és a retrieved adatokat.

Pattern 5: Tool & Capability Management:

  • MCP (Model Context Protocol): Standard az eszközök csatlakoztatására.
  • Javaslat: Az eszközök számát érdemes ágensenként 20 alatt tartani a pontosság érdekében.
